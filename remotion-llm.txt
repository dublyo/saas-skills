Purpose
Use this file as the local AI reference before planning or coding with Remotion.
It is a practical summary of the official Remotion API docs plus implementation
tips for a SaaS/video-rendering workflow. Keep the official links in this file
as the source of truth when exact option names or version behavior matter.

Core Rule
Remotion is React rendered on a deterministic video timeline. Treat every visual
state as a function of props, frame number, fps, and loaded assets. Do not rely
on browser time, CSS transitions, CSS animations, runtime randomness, or user
interaction during a render.

How To Use This Skill
1. First identify whether the task is preview, composition authoring, local
   render, self-hosted server render, Lambda/Cloud Run/Vercel render, or Player
   embed. The correct APIs differ.
2. Check the existing project package manager and Remotion version before adding
   packages or using recently added APIs.
3. Prefer the smallest Remotion package that solves the problem. Do not add
   Lambda, Cloud Run, WebCodecs, Skia, Three, or Studio APIs unless the feature
   actually needs them.
4. For production rendering, separate authoring code from worker orchestration:
   the Remotion project renders frames; the backend owns queueing, retries,
   progress, storage, cancellation, authentication, and billing.
5. When exact API options matter, open the linked official page. This file is
   the working map, not a replacement for the docs.

Project Setup And Preview
- New blank project: use `npx create-video@latest --yes --blank --no-tailwind my-video`
  unless the existing repo already has a Remotion app.
- Start Studio: `npx remotion studio`.
- Run a cheap visual smoke test with one still frame, for example:
  `npx remotion still <composition-id> --scale=0.25 --frame=30`.
- At 30 fps, frame 30 is the 1 second mark because frames are zero-based.
- Use `npx remotion render` for full local renders only after a still/frame check
  catches obvious layout and asset failures.

Composition Model
- Register the root with `registerRoot()`: https://www.remotion.dev/docs/register-root
- Define videos with `<Composition>`: https://www.remotion.dev/docs/composition
- Define single-frame outputs with `<Still>`: https://www.remotion.dev/docs/still
- Use `<Folder>` only for Studio organization: https://www.remotion.dev/docs/folder
- A composition declares `id`, component, `durationInFrames`, `fps`, `width`,
  and `height`. Its props should be typed and serializable.
- Put compositions in the Remotion root, normally `src/Root.tsx`.
- Use `defaultProps` for Studio defaults and initial render props.
- Prefer TypeScript `type` for composition props when tying `defaultProps` to
  component props.
- Use stable composition IDs. Treat them like API identifiers because render
  workers, queues, and user projects may store them.

Typed Inputs And Studio Controls
- Use Zod schemas for user-editable composition props.
- The schema top level must be a `z.object()` because React component props are
  an object.
- Use `@remotion/zod-types` for richer Studio controls:
  - `zColor()`: https://www.remotion.dev/docs/zod-types/z-color
  - `zTextarea()`: https://www.remotion.dev/docs/zod-types/z-textarea
  - `zMatrix()`: https://www.remotion.dev/docs/zod-types/z-matrix
- Use `Interactive.withSchema()` or `InteractivitySchema` when exposing timeline
  controls for components/effects:
  - https://www.remotion.dev/docs/interactive-with-schema
  - https://www.remotion.dev/docs/interactivity-schema
- Validate AI-generated JSON before passing it to Remotion. Reject or repair
  invalid scene duration, asset URL, color, font, and transition values before
  the render job starts.

Dynamic Metadata
- Use `calculateMetadata` when duration, fps, dimensions, props, output name, or
  codec defaults depend on input data.
- Good uses: calculate duration from narration/audio/video, fetch project data,
  normalize props, set `defaultOutName`, select dimensions by platform.
- Use the provided `abortSignal` in async fetches so Studio can cancel stale work.
- Keep `durationInFrames` placeholders sensible because Studio still needs a
  starting value before dynamic metadata resolves.
- Source: https://www.remotion.dev/docs/composition

Input Props
- Prefer receiving props directly in the composition component.
- Use `getInputProps()` only when the root needs access to CLI or Node API props.
- `getInputProps()` is not available inside Player/client-side rendering; pass
  props to `<Player inputProps={...}>` instead.
- Source: https://www.remotion.dev/docs/get-input-props

Environment Awareness
- Use `getRemotionEnvironment()` or the newer environment hook when behavior must
  differ between Studio, Player, rendering, or server-side contexts.
- Do not branch visual output in a way that makes preview and render disagree.
- Safe branches: logging, debug overlays, disabled expensive editor-only controls.
- Source: https://www.remotion.dev/docs/get-remotion-environment

Timeline And Frame Rules
- `useCurrentFrame()` returns the current frame. It is zero-based.
- Inside a `<Sequence>`, `useCurrentFrame()` is local to that sequence.
- Use `useVideoConfig()` for `fps`, `durationInFrames`, `width`, and `height`.
- Express human durations in seconds in your thinking, then convert with `seconds * fps`.
- Store scene plans in seconds if that is more natural for AI/product data, but
  convert to integer frames at the Remotion boundary.
- Sources:
  - https://www.remotion.dev/docs/use-current-frame
  - https://www.remotion.dev/docs/use-video-config

Sequencing
- Use `<Sequence>` to delay, trim, or isolate timeline sections.
- Use `<Series>` when scenes should play one after another.
- Use negative `offset` in `<Series.Sequence>` only when scenes intentionally
  overlap.
- `<Sequence>` wraps children in absolute fill layout by default. Use `layout="none"`
  when the wrapper would break layout.
- Premount expensive or media-heavy scenes before they appear when needed.
- `<Freeze>` is useful for holding a component at one frame.
- `<Loop>` is useful for repeated content but make sure looped content has a
  defined duration.
- Sources:
  - https://www.remotion.dev/docs/sequence
  - https://www.remotion.dev/docs/series
  - https://www.remotion.dev/docs/freeze
  - https://www.remotion.dev/docs/loop

Animation Rules
- All render-critical animation must be driven by `useCurrentFrame()`.
- Do not use CSS transitions, CSS keyframes, Tailwind `animate-*`, or Tailwind
  `transition-*` for video motion.
- Use `interpolate()` with explicit frame ranges and both extrapolation clamps
  for most UI/video motion.
- Use `Easing.bezier(x1, y1, x2, y2)` when matching design curves or CSS
  cubic-bezier timing.
- Use `spring()` for physics-like motion and `measureSpring()` when duration must
  be known.
- Use `interpolateColors()` for color transitions.
- Normalize complex motion to one progress value, then derive opacity, transform,
  blur, and scale from that progress.
- Sources:
  - https://www.remotion.dev/docs/interpolate
  - https://www.remotion.dev/docs/easing
  - https://www.remotion.dev/docs/spring
  - https://www.remotion.dev/docs/measure-spring
  - https://www.remotion.dev/docs/interpolate-colors
  - https://www.remotion.dev/docs/animation-utils/make-transform
  - https://www.remotion.dev/docs/animation-utils/interpolate-styles

Layout
- Use `<AbsoluteFill>` for full-frame layers:
  https://www.remotion.dev/docs/absolute-fill
- Use fixed composition dimensions and make child layout deterministic.
- Avoid layout that depends on viewport size outside the composition dimensions.
- For text-heavy generated videos, use `@remotion/layout-utils`:
  - `measureText()`: https://www.remotion.dev/docs/layout-utils/measure-text
  - `fitText()`: https://www.remotion.dev/docs/layout-utils/fit-text
  - `fitTextOnNLines()`: https://www.remotion.dev/docs/layout-utils/fit-text-on-n-lines
  - `fillTextBox()`: https://www.remotion.dev/docs/layout-utils/fill-text-box
- Load fonts before measuring text and use the same font family, size, weight,
  letter spacing, and style for measuring and rendering.
- For debug outlines, prefer `outline` over `border` because borders alter layout.

Assets
- Put local assets in the project `public/` folder next to the package.json that
  owns Remotion.
- Use `staticFile("path.ext")` for files in `public/`; do not use raw relative
  paths for render assets.
- Remote URLs can be used directly when CORS, availability, and auth are handled.
- Prefer Remotion-aware components because they wait for assets and reduce
  flicker/missing frames:
  - `<Img>`: https://www.remotion.dev/docs/img
  - `<AnimatedImage>`: https://www.remotion.dev/docs/animatedimage
  - `<OffthreadVideo>`: https://www.remotion.dev/docs/offthreadvideo
  - `@remotion/media` `<Video>`: https://www.remotion.dev/docs/media/video
  - `@remotion/media` `<Audio>`: https://www.remotion.dev/docs/media/audio
- Source for static files: https://www.remotion.dev/docs/staticfile

Images
- Use `<Img>` instead of plain `<img>` for render-critical images.
- `<Img>` waits for images before rendering frames and supports retry/error
  behavior.
- For image effects, use `<CanvasImage>` and `createEffect()` when a canvas-based
  transform is appropriate:
  - https://www.remotion.dev/docs/canvasimage
  - https://www.remotion.dev/docs/create-effect

Video And Audio
- Prefer `@remotion/media` `<Video>` and `<Audio>` for modern media workflows,
  especially where Player buffering or browser/WebCodecs behavior matters.
- `<OffthreadVideo>` uses FFmpeg frame extraction during rendering. It is useful
  for exact server-side video frames but is not supported by `@remotion/web-renderer`.
- `<Html5Video>` and `<Html5Audio>` are lower-level synchronized HTML media tags.
- Treat trim/start/end/duration props as frame-based unless the specific utility
  explicitly returns seconds. For "2 seconds", use `2 * fps`.
- Reverse playback is not generally supported.
- Dynamic volume callbacks receive a media-local frame value that starts when the
  media starts, not necessarily the absolute composition frame.
- Pitch/tone changes can be server-render-only depending on component and API;
  verify in the current docs before promising Player parity.
- Sources:
  - https://www.remotion.dev/docs/media/video
  - https://www.remotion.dev/docs/media/audio
  - https://www.remotion.dev/docs/offthreadvideo
  - https://www.remotion.dev/docs/html5-video
  - https://www.remotion.dev/docs/html5-audio

Tailwind
- Tailwind is fine for static styling in Remotion after it is enabled.
- Never use Tailwind animation or transition utilities for render-critical motion.
- Use frame math plus normal class/style output instead.
- Enable Tailwind with the Remotion docs that match the project version:
  - Tailwind current: https://www.remotion.dev/docs/tailwind/enable-tailwind
  - Tailwind v4: https://www.remotion.dev/docs/tailwind-v4/enable-tailwind

Async Work During Rendering
- Use `delayRender()` to block rendering for async work inside components.
- Always call `continueRender(handle)` when work succeeds.
- Call `cancelRender(error)` when the render cannot recover.
- Prefer `calculateMetadata` for render-critical data that affects duration,
  dimensions, or final props.
- Add labels/timeouts for easier debugging when a render hangs.
- Clean up pending async work in React effects so stale renders do not continue.
- Sources:
  - https://www.remotion.dev/docs/delay-render
  - https://www.remotion.dev/docs/continue-render
  - https://www.remotion.dev/docs/cancel-render

Transitions
- Use `@remotion/transitions` when scenes need declarative transitions.
- `<TransitionSeries>` is a scene series with transition and overlay slots.
- Transitions overlap neighboring scenes and shorten total duration by the
  transition duration.
- Overlays sit over the cut point and do not shorten the total duration.
- Do not put overlay next to overlay or transition next to overlay without
  checking the current rules.
- Use `linearTiming()` for fixed-duration transitions and `springTiming()` when
  spring physics are desired.
- Useful links:

Player Embed
- Use `@remotion/player` to embed a Remotion composition inside a normal React
  app, for example Next.js or Vite.
- `<Player>` needs component/lazy component, duration, fps, composition width,
  composition height, and input props.
- `<Player>` is for preview/playback in the web app, not for server rendering.
- Use `<Thumbnail>` for still previews.
- Do not use `getInputProps()` inside Player. Pass props via `inputProps`.
- Links:
  - https://www.remotion.dev/docs/player/player
  - https://www.remotion.dev/docs/player/thumbnail

Bundling And Local/Server Rendering
- Use `bundle()` from `@remotion/bundler` to prepare a Remotion project for Node
  rendering.
- Important trick: call `bundle()` only when source code changes. Reuse the same
  bundle for many renders by changing `inputProps`.
- Use `getCompositions()` when you need the list of available compositions.
- Use `selectComposition()` when rendering one known composition; it evaluates
  that composition and its metadata.
- Use `renderMedia()` for video/audio renders.
- Use `renderStill()` for one image.
- Use `renderFrames()` and `stitchFramesToVideo()` only when you need explicit
  frame-level processing or custom assembly.
- Reuse a browser with `openBrowser()` for multiple renders when worker lifetime
  allows it.
- Use `ensureBrowser()` during worker setup to make Chrome availability explicit.
- Use `makeCancelSignal()` for user cancellation and backend timeouts.
- Use `ensureFfmpeg()` and `ensureFfprobe()` when the worker depends on local
  media tooling.

Self-Hosted Worker Pattern
Use this pattern for a SaaS that renders on an Ubuntu CPU server:
1. Backend validates user input and stores a render job.
2. AI pipeline produces a strict JSON scene plan.
3. Backend validates the JSON against a schema and normalizes seconds to frames.
4. Worker receives job ID and input props from Redis/BullMQ or another queue.
5. Worker ensures browser/ffmpeg dependencies are available.
6. Worker bundles only when source changed, or reuses a cached bundle.
7. Worker selects the composition with the validated props.
8. Worker renders with progress logging and cancellation support.
9. Worker uploads output to object storage and stores metadata.
10. Backend exposes status, progress, logs, preview/still, and download URL.

Do not describe self-hosted Remotion as a hosted rendering service. On a user's
own server, the app must own queueing, worker capacity, retries, logs, progress,
cleanup, browser lifecycle, storage, and security.

AI Video Product JSON Contract Tips
- Keep the AI contract explicit: compositionId, fps, width, height, total seconds,
  scenes, assets, narration, captions, transitions, color tokens, font tokens.
- Do not let AI choose arbitrary React component names or import paths.
- Map AI scene types to a finite renderer registry, for example:
  `title`, `image`, `stat`, `quote`, `timeline`, `comparison`, `caption`.
- Validate each scene duration and ensure the sum matches the composition duration
  or the dynamic metadata result.
- Validate remote asset URLs before rendering. Cache or mirror important assets
  to object storage for reliable renders.
- Generate one still/preview frame before a full render when jobs are expensive.
- Store render inputs and exact package/version metadata for reproducibility.
- Use deterministic random seeds if a template needs variation.

Common Mistakes To Avoid
- Using CSS animations or transitions and expecting them to render correctly.
- Using Tailwind `animate-*` or `transition-*` classes for video motion.
- Passing raw `public/foo.png` strings instead of `staticFile("foo.png")`.
- Fetching live network data on every frame.
- Calling `bundle()` for every job even when source code did not change.
- Using `getCompositions()` for one render when `selectComposition()` is enough.
- Forgetting that frames inside `<Sequence>` are local.
- Letting transitions shorten duration without updating total duration.
- Measuring text before fonts are loaded.
- Assuming Player behavior is identical to server rendering.
- Assuming pitch, media decoding, or browser APIs behave the same in Studio,
  Player, Lambda, local Node, and Cloud Run.
- Letting user input reach file paths, output locations, shell commands, or
  remote URLs without validation.
- Treating render success as only "function returned"; verify output exists,
  has nonzero size, expected duration, expected dimensions, and playable media.

Security Checklist
- Validate all input props with Zod or equivalent backend schema.
- Sanitize URLs and only allow expected protocols/domains when users submit media.
- Do not pass user strings into shell commands.
- Use isolated temp directories per job.
- Prevent path traversal in output and static file writes.
- Limit render duration, dimensions, concurrency, and media file size.
- Add cancellation/timeouts for stalled renders.
- Store secrets only in backend/worker env, never in composition props shown to
  Studio or Player.
- For Lambda webhooks, validate signatures with the official API.
- Do not expose presigned/download URLs longer than the product requires.

Testing And Verification
- Compile/typecheck the Remotion project.
- Start Studio for authoring issues.
- Render a still at an early frame, middle frame, and final-frame-minus-one.
- Render a short low-scale video before a full production render.
- Test with missing asset, slow asset, malformed props, long text, long captions,
  very short scenes, and unsupported media.
- For backend workers, verify queue state transitions: queued, rendering,
  succeeded, failed, cancelled.
- Verify retry behavior is idempotent and does not leak temp files or duplicate
  billing/storage records.
- Verify output metadata with ffprobe or Remotion renderer metadata helpers.

API Selection Cheat Sheet
- Need full-frame layer: `<AbsoluteFill>`.
- Need composition definition: `<Composition>`.
- Need still image output: `<Still>`.
- Need delay/trim timeline section: `<Sequence>`.
- Need back-to-back scenes: `<Series>`.
- Need scene transitions: `<TransitionSeries>`.
- Need current timeline position: `useCurrentFrame()`.
- Need fps/dimensions/duration: `useVideoConfig()`.
- Need value mapping over time: `interpolate()`.
- Need color mapping: `interpolateColors()`.
- Need physics motion: `spring()` plus `measureSpring()` if duration matters.
- Need local asset URL: `staticFile()`.
- Need render-aware image: `<Img>`.
- Need exact server-side video frames: `<OffthreadVideo>` or `@remotion/media`
  depending on runtime requirements.
- Need modern media component: `@remotion/media` `<Video>` / `<Audio>`.
- Need async component wait: `delayRender()` / `continueRender()` / `cancelRender()`.
- Need dynamic duration/props/dimensions: `calculateMetadata`.
- Need server render: `bundle()` -> `selectComposition()` -> `renderMedia()`.
- Need composition list: `getCompositions()`.
- Need browser reuse: `openBrowser()`.
- Need browser installed: `ensureBrowser()`.
- Need cancellation: `makeCancelSignal()`.
- Need web app preview: `@remotion/player` `<Player>`.
- Need captions: `@remotion/captions`.
- Need fitted text: `@remotion/layout-utils`.
- Need Studio prop controls: Zod schema plus `@remotion/zod-types`.

Recommended Self-Hosted Render Flow
Pseudo-flow only; verify exact options in the linked docs for the installed
Remotion version.

1. Validate input.
2. Build or load cached bundle with `bundle(entryPoint)`.
3. Select composition:
   `selectComposition({ serveUrl, id, inputProps })`.
4. Render:
   `renderMedia({ serveUrl, composition, codec, outputLocation, inputProps })`.
5. Record progress, logs, output path, duration, size, and errors.
6. Upload output to storage and clean temporary files.

Important source links:
- Bundle once, render many input props: https://www.remotion.dev/docs/bundle
- Select one composition: https://www.remotion.dev/docs/renderer/select-composition
- Render media: https://www.remotion.dev/docs/renderer/render-media
