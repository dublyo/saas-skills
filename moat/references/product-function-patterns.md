## Pattern Selection

Pick patterns based on the SaaS motion:

- Self-serve SMB: free tool, activation loop, lightweight workflow, referral/share, SEO/GEO, templates.
- Team SaaS: collaboration, roles, approvals, comments, notifications, audit history.
- Vertical SaaS: system of record, domain workflow, compliance, integrations, proprietary data.
- Agency SaaS: client workspaces, white-label reports, share links, reseller billing, branded portal.
- API-first SaaS: developer experience, API keys, logs, webhooks, SDKs, usage limits, reliability.
- Enterprise SaaS: SSO, SCIM, audit logs, procurement, compliance, admin controls, SLA.
- AI SaaS: workflow ownership, proprietary feedback, human review, data permissions, cost controls.

## System Of Record

Goal:

- Become the place where operational truth lives.

Functions:

- Core records.
- Ownership.
- History.
- Attachments.
- Comments.
- Decisions.
- State transitions.
- Audit logs.
- Permissions.
- Search.
- Reports.
- Exports.
- Retention policy.

Moat created:

- Switching costs and workflow depth.

Profit protected:

- GRR, expansion, pricing power.

Tests:

- Can a customer run the weekly/monthly business process without this product?
- Does the product store the history needed for audits, reports, renewal, or decisions?

## Workflow Lock-In

Goal:

- Own a repeated business process, not just a page.

Functions:

- Step-by-step flow.
- Assignment.
- Status.
- Approval.
- Due dates.
- SLAs.
- Notifications.
- Escalation.
- Automation.
- Templates.
- Recurring schedules.

Moat created:

- Process power and switching costs.

Profit protected:

- Retention and seat expansion.

Tests:

- What breaks operationally if the customer cancels?
- How many users touch this workflow weekly?

## Collaboration And Multi-Player

Goal:

- Make the product more valuable as more people participate.

Functions:

- Team invites.
- Roles.
- Shared workspaces.
- Comments.
- Mentions.
- Client portals.
- Shared reports.
- Approval chains.
- Notifications.
- Presence/activity.

Moat created:

- Network and switching-cost effects inside the account.

Profit protected:

- Expansion, seat growth, retention.

Tests:

- Does inviting another person increase the value for existing users?
- Does the product create a collaboration history that would be painful to lose?

## Data Feedback Loop

Goal:

- Make the product better with use.

Functions:

- Event tracking.
- Outcome capture.
- Labels.
- Corrections.
- Human review.
- Benchmarking.
- Data quality checks.
- Model or rules improvement.
- Customer-specific learning.
- Global anonymized learning when permitted.

Moat created:

- Data advantage, process power, trust.

Profit protected:

- Better retention, pricing power, lower service cost.

Tests:

- Does more usage improve output quality?
- Is the data proprietary, fresh, permissioned, and hard to reconstruct?

## Integrations And APIs

Goal:

- Embed the product into the customer's operating environment.

Functions:

- OAuth.
- Native integrations.
- API keys.
- Webhooks.
- Sync logs.
- Field mapping.
- Error recovery.
- Retry controls.
- Integration health.
- Admin permissions.

Moat created:

- Switching costs and enterprise readiness.

Profit protected:

- Retention, ACV, expansion.

Tests:

- Would removing the product break other systems?
- Are integration failures visible and recoverable?

## Public/Shareable Artifact

Goal:

- Turn product output into distribution.

Functions:

- Share links.
- Public/private controls.
- Public reports.
- Branded exports.
- Embed widgets.
- Badges.
- Viewer analytics.
- Signup from shared page.
- Expiration and revocation.

Moat created:

- Distribution, brand, product virality.

Profit protected:

- Lower CAC, referrals, expansion.

Tests:

- Does sharing create value for the recipient?
- Does it expose enough value to drive signup without leaking private data?

## Marketplace Or Ecosystem

Goal:

- Let third parties add value.

Functions:

- App marketplace.
- Templates.
- Partner directory.
- Developer docs.
- Webhooks.
- API.
- Revenue share.
- Certification.
- Reviews.
- Install flow.

Moat created:

- Network effects and ecosystem lock-in.

Profit protected:

- Lower CAC, expansion, retention.

Tests:

- Why would partners build here instead of elsewhere?
- Does every new partner increase customer value?

## Agency, Reseller, And White-Label

Goal:

- Make partners distribute and embed the product.

Functions:

- Partner account.
- Client workspaces.
- White-label branding.
- Custom domain.
- Branded reports.
- Client permissions.
- Reseller billing.
- Margin controls.
- Partner analytics.
- Client invitations.

Moat created:

- Distribution advantage, switching costs, process ownership.

Profit protected:

- CAC, payback, partner retention, ARPA.

Tests:

- Would an agency lose process, client trust, or branded assets if they switched?
- Are reseller margins protected by usage limits?

## Compliance And Trust

Goal:

- Reduce buyer risk and increase willingness to pay.

Functions:

- Audit logs.
- SSO.
- SCIM.
- Admin controls.
- Data retention.
- Data export.
- DPA.
- Subprocessor list.
- Status page.
- Incident response.
- Security reports.
- Access review.

Moat created:

- Trust, procurement, compliance, enterprise switching costs.

Profit protected:

- ACV, close rate, renewal, enterprise access.

Tests:

- Which customer segment will pay more because risk is lower?
- Which compliance feature is required versus premature enterprise baggage?

## AI Workflow Moat

Goal:

- Use AI to own outcomes, not only generate text.

Functions:

- Domain-specific input schema.
- Retrieval from trusted customer context.
- Human approval.
- Feedback capture.
- Action execution.
- Explainability.
- Audit trail.
- Confidence thresholds.
- Model routing.
- Cost limits.
- Evaluation set.
- Failure handling.

Moat created:

- Workflow depth, data feedback, trust, cost advantage.

Profit protected:

- Higher willingness to pay, retention, lower service cost, gross margin.

Tests:

- Does AI complete or improve a real business process?
- Does customer feedback make the system better?
- Can a generic chatbot replace this?

## Cost Advantage Functions

Goal:

- Keep value delivery profitable.

Functions:

- Usage metering.
- Quotas.
- Rate limits.
- Abuse detection.
- Cost dashboard.
- Plan-level limits.
- Caching.
- Batch jobs.
- Model routing.
- Async processing.
- Self-serve onboarding.
- Support automation.

Moat created:

- Cost advantage and margin protection.

Profit protected:

- Gross margin and contribution margin.

Tests:

- Which features can create negative margin?
- Can heavy users be priced or limited without harming good customers?

## Anti-Patterns

Avoid:

- Building all moats at once.
- Adding enterprise friction before enterprise customers.
- Creating fake switching costs through bad export or lock-in.
- Making data export impossible as a retention tactic.
- Claiming data moat without outcome improvement.
- Claiming network effect without increasing value per participant.
- Claiming AI moat without proprietary workflow, data, or trust.
- Letting high-usage customers destroy margin.
- Building integrations that are not tied to retention or acquisition.
- Treating founder sales as permanent distribution moat.
